## 4. Conclusion
- **offload_launch ж”¶зӣҠжҳҫи‘—гҖӮ** L1 еңЁ 1024/2048/4096 дёҠдёӢйҷҚ 35вҖ“51%пјҲ2048пјҡ7.19 вҶ’ 3.96 msпјүгҖӮдё»иҰҒжқҘиҮӘ `_build_slot_mappings` L2 дёӢйҷҚ 44вҖ“74%пјҢд»ҘеҸҠ `put_async` L2 дёӢйҷҚзәҰ 17вҖ“19%пјҲslot е·ІеңЁ CPUпјҢзңҒеҺ» loop еҶ…е…ЁйҮҸ `.to("cpu")`пјүгҖӮ
- **onboard_launchпјҡзңӢ L1 еҸҳеҢ–дёҚеӨ§пјҢзңӢ slot еӯҗйҳ¶ж®ө optimized з•ҘдјҳгҖӮ** `_build_slot_mappings` L2 еңЁ 2048 дёҠ 0.62 вҶ’ 0.95 msпјҲ+53%пјүпјҢдё»еӣ жҳҜ baseline зҡ„ sync/D2H иҗҪеңЁ **жңӘеҚ•зӢ¬жү“ NVTX зҡ„ loop д»Јз Ғ** йҮҢпјҲ`slot_d2h` вүҲ 0.32 msпјүгҖӮеӯҗйҳ¶ж®өеҗҲи®Ўпјҡbaseline build+`slot_d2h` вүҲ 1.43 msпјҢoptimized `page_ids_d2h`+`cpu_expand` вүҲ 1.02 msгҖӮ
- **з»§з»ӯдјҳеҢ–пјҡ** йҖҡиҝҮеҲҶжһҗ timelineеҸ‘зҺ°пјҢv1 иҷҪжҠҠ D2H йҮҸд»Һж•ҙж®ө slot еҺӢеҲ° `page_ids`пјҢдҪҶе®һзҺ°д»ҚжҳҜ **жҜҸдёӘ seq еҗ„еҒҡдёҖж¬Ў `page_ids.to(cpu)`**пјҲ8 ж¬Ўе°Ҹ D2H + еӨҡж¬Ў syncпјүпјҢдё”еңЁ CPU дёҠз”Ё **numpy йҖҗ page `arange` жӢјжҺҘ** еұ•ејҖ slotгҖӮеӯҗйҳ¶ж®ө profilingпјҲ`page_ids_d2h` + `cpu_expand`пјүжҳҫзӨәпјҡ`cpu_expand` еңЁ 2048 дёҠзәҰ **0.76 ms/жӯҘ**пјҢеҚ  build еҶ…еӨ§еӨҙпјӣ8 ж¬Ў per-seq D2H зҡ„ API/sync ејҖй”Җд№ҹеҒҸй«ҳгҖӮеӣ жӯӨ v2 ж”№дёә **ж•ҙжү№ `kv_indices`/`kv_indptr` дёҖж¬Ў D2H**пјҢе№¶з”Ё **torch CPU е№ҝж’ӯ** еҒҡеҗ‘йҮҸеҢ– expandпјҢиҜҰи§Ғ В§5гҖӮ

### 5.3 ConclusionпјҲv2пјү

- **`_build_slot_mappings` L2 еңЁ v2 дёҠзЁіе®ҡдҪҺдәҺ v1**пјҲ2048 offloadпјҡ1.13 вҶ’ 0.49 msпјӣonboardпјҡ0.95 вҶ’ 0.33 msпјүгҖӮbulk D2H + еҗ‘йҮҸеҢ– expand иҫҫеҲ°йў„жңҹгҖӮ
- **offload_launch L1**пјҡv2 зӣёеҜ№ Before йҷҚ **46вҖ“53%**пјӣзӣёеҜ№ v1пјҢ4096 еҶҚйҷҚ 16%пјҢ1024/2048 дёҺ v1 еҹәжң¬жҢҒе№ігҖӮ
- **onboard_launch L1**пјҡv2 зӣёеҜ№ v1 еңЁ 2048/4096 дёҠйҷҚ **15вҖ“32%**пјҢзӣёеҜ№ Before з•ҘдјҳпјҲзәҰ -5%пјүпјӣL1 д»ҚеҸ— `launch` зӯүеӯҗйҳ¶ж®өжіўеҠЁеҪұе“ҚпјҢдҪҶ build и·Ҝеҫ„е·Іжҳҫи‘—еҝ«дәҺ v1гҖӮ
